### 1. FILE ANALYSIS — Enhancement

**Yang sudah ada:** magic bytes detection, binwalk, file carving (foremost), strings, header repair

**Saran tambahan:**

**1.1 Deep File Signature Database**
Perluas MAGIC_MAP dari ~25 signature menjadi 100+ signature, termasuk format-format CTF yang jarang tapi sering muncul seperti LUKS encrypted volume, SquashFS, ext2/3/4 filesystem images, Android sparse image, Java KeyStore, PCAPNG dengan custom options block, dan berbagai format audio/video container.

**1.2 Chunk-Based File Parser**
Tambahkan parser khusus untuk format yang berbasis chunk seperti PNG (baca tiap chunk IDAT, tEXt, zTXt, iTXt, eXIf secara individual), RIFF/WAV (LIST chunk, INFO chunk), IFF, dan JPEG markers (APP0 sampai APPn, COM marker). Sering di CTF flag disembunyikan di custom chunk yang valid secara struktur tapi tidak ditampilkan viewer biasa.

**1.3 File-within-File Detection**
Deteksi polyglot file secara lebih agresif, bukan hanya cek satu magic bytes di awal. Scan seluruh body file untuk magic bytes tersembunyi di offset acak, kemudian ekstrak dan analisis secara rekursif. Bedakan antara appended data, prepended data, dan injected data di tengah file.

**1.4 Entropy Map Visualization**
Generate entropy heatmap per 512-byte block dari seluruh file, simpan sebagai gambar grayscale. Region entropy tinggi (>7.5) menandakan encrypted/compressed data, region entropy rendah (<1.0) menandakan null padding atau repetisi. Output berupa file PNG yang bisa langsung dilihat analyst.

**1.5 File Timeline Analysis**
Untuk file yang memiliki embedded timestamp (EXIF, ZIP central directory, NTFS MFT, PDF metadata), ekstrak semua timestamp dan urutkan secara kronologis. Anomali timestamp (file dibuat di masa depan, atau epoch 0) sering jadi petunjuk CTF.

### 2. STEGANOGRAPHY — Enhancement Besar

**Yang sudah ada:** zsteg, steghide, stegseek, outguess, LSB extract, bit planes, color remapping, WAV LSB

**Saran tambahan:**

**2.1 Audio Steganography Suite**
Saat ini hanya ada WAV LSB yang sangat basic. Perlu ditambahkan:
- Spectrogram generator untuk WAV/MP3/FLAC menggunakan scipy/matplotlib, karena banyak CTF menyembunyikan flag sebagai gambar dalam spektrum frekuensi audio
- Phase encoding detector: bandingkan phase antar frame untuk mendeteksi data tersembunyi
- Echo hiding detector: analisis auto-korelasi untuk menemukan echo yang ditanamkan
- MP3 steganography via MP3Stego tool wrapper
- DTMF tone decoder: decode tone telepon dari audio

**2.2 Advanced Image Steganography**
- DCT coefficient analysis untuk JPEG: flag sering disembunyikan di koefisien DCT tertentu, bukan di domain spasial
- Palette-based steganography untuk GIF dan PNG-8: urutan palette entry bisa encode pesan
- Chi-square attack implementation: deteksi LSB stego secara statistik menggunakan chi-square test, lebih akurat dari sekedar melihat LSB ratio
- RS (Regular-Singular) analysis: metode steganalysis yang lebih sophisticated
- Pixel value differencing (PVD) decoder
- F5 algorithm decoder untuk JPEG (lebih advanced dari outguess)
- JSteg decoder (berbeda dengan outguess, menggunakan DCT coefficients)

**2.3 Document Steganography**
- White-space steganography di file teks: tab vs spasi, trailing whitespace per baris bisa encode binary
- Font-based steganography di PDF/DOCX: perubahan ukuran font 0.1pt yang tidak terlihat
- Word/line spacing manipulation di dokumen
- OpenDocument format (ODT/ODS) hidden content dalam XML stream

**2.4 Network Steganography**
- IP ID field steganography dalam PCAP
- TCP sequence number analysis untuk data tersembunyi
- DNS TTL value encoding
- ICMP payload steganography
- HTTP header order steganography (Tor fingerprinting technique)

**2.5 Stegsolve Equivalent (Pure Python)**
Buat implementasi fitur stegsolve tanpa Java dependency: semua filter warna (red plane 0-7, green plane 0-7, blue plane 0-7), XOR combinator antara dua gambar, image addition/subtraction, random color map, full alpha analysis.

### 3. MEMORY & DISK FORENSICS — Enhancement

**Yang sudah ada:** Volatility 3 wrapper, NTFS deleted file recovery, partition table analysis, MBR detection

**Saran tambahan:**

**3.1 Volatility Plugin Expansion**
Saat ini hanya menjalankan beberapa plugin standar. Perlu ditambahkan:
- windows.modscan dan windows.modules untuk kernel module yang di-inject
- windows.malware.malfind dengan yara rule custom untuk CTF
- windows.pstree dengan deteksi anomali parent-child (svchost.exe seharusnya child dari services.exe)
- windows.handles untuk mencari file handle yang masih terbuka
- windows.clipboard untuk isi clipboard
- windows.consoles untuk command history di cmd.exe
- linux.psaux dengan full command line argument
- Automatic profile detection untuk Volatility 2 backward compatibility

**3.2 Autopsy/TSK Deep Integration**
Saat ini hanya pakai mmls/fls/icat yang basic. Tambahkan:
- Timeline generation menggunakan mactime (The Sleuth Kit)
- $MFT parser langsung tanpa mount: parse Master File Table untuk file yang sudah dihapus tapi MFT entry belum di-overwrite
- $LogFile analysis untuk Windows journal
- $USNJournal parsing untuk track file changes
- Volume Shadow Copy detection dan extraction
- Alternate Data Streams (ADS) detection di NTFS: flag sering disembunyikan di ADS

**3.3 Memory Artifact Extraction**
- Extract credentials dari memory: LSASS dump analysis untuk NTLM hash
- Browser artifact extraction: Chrome/Firefox password, cookie, history dari memory
- Clipboard content extraction
- Screenshot reconstruction dari framebuffer memory
- Encryption key detection: scan untuk pola key schedule AES, RSA private key material

**3.4 Disk Image Format Support**
Saat ini hanya dd/raw. Tambahkan support:
- E01/EWF (EnCase) format via libewf atau ewf-tools
- VMDK flat dan sparse
- VHD/VHDX
- QCOW2
- DMG (macOS disk image)
- Automatic format detection dan conversion ke raw sebelum analisis

**3.5 Anti-Forensics Detection**
- Timestamp stomping detection: bandingkan $STANDARD_INFORMATION vs $FILE_NAME timestamp
- Log clearing detection: Windows event log gaps
- Wipe tool artifact: pola overwrite dari tools seperti CCleaner, Eraser
- Hibernation file (hiberfil.sys) analysis
- Pagefile.sys analysis untuk recover plaintext dari encrypted files

### 4. NETWORK FORENSICS — Enhancement Besar

**Yang sudah ada:** tshark PCAP analysis, HTTP object extraction, DNS query extraction, credential extraction, TCP stream reconstruction, DNS tunneling detector, per-packet payload decoder

**Saran tambahan:**

**4.1 Protocol-Specific Decoders**
Saat ini hanya handle HTTP dan DNS dengan baik. Perlu ditambahkan:
- FTP full session reconstructor: gabungkan control channel dan data channel, reconstruct file yang di-transfer
- SMTP/POP3/IMAP email reconstructor: extract attachment, decode MIME
- SMB/CIFS file share reconstructor: carve file yang di-transfer via SMB
- Telnet/SSH session reconstructor (untuk Telnet yang tidak encrypted)
- IRC message extractor
- TLS/SSL handshake analyzer: extract certificate, detect weak cipher
- MQTT topic/message extractor untuk IoT CTF
- Modbus/DNP3 untuk ICS/SCADA CTF

**4.2 Traffic Pattern Analysis**
- Beaconing detection: analisis interval request yang periodic (malware C2)
- Data exfiltration pattern: volume data yang keluar lebih besar dari yang masuk
- Port scan pattern: deteksi SYN scan, UDP scan
- Covert channel detection: timing analysis, size analysis
- TOR traffic detection: circuit establishment pattern

**4.3 File Carving dari PCAP**
Saat ini hanya extract HTTP objects. Tambahkan:
- Carve file dari semua protokol: FTP-DATA, SMB, TFTP
- Reassemble fragmented transfers
- Extract file dari raw TCP stream tanpa protocol parsing
- MIME type detection untuk extracted files

**4.4 Wireless/802.11 Analysis**
- WEP/WPA handshake extraction untuk cracking
- Probe request SSID list (device history)
- Deauth attack detection
- Hidden SSID reveal dari probe response
- MAC address vendor lookup

**4.5 PCAP Differential Analysis**
Bandingkan dua PCAP file untuk menemukan perbedaan: request yang ada di satu file tapi tidak di yang lain, response code yang berubah. Berguna untuk before/after analysis di incident response CTF.

### 6. CRYPTOGRAPHY — Enhancement

**Yang sudah ada:** RSA (weak prime, Fermat, Common Modulus, Bellcore), Vigenere, XOR KPA, Classic Cipher, Encoding Chain, AES-CBC Padding Oracle

**Saran tambahan:**

**6.1 RSA Enhancement**
- Wiener's attack untuk small private exponent (d kecil)
- Håstad's broadcast attack: pesan sama dienkripsi dengan N berbeda dan e=3
- Boneh-Durfee attack (extension dari Wiener untuk d lebih besar)
- Franklin-Reiter related message attack
- Partial key exposure attack
- Integration dengan FactorDB API: cek apakah N sudah pernah difaktorisasi
- Integration dengan online RSA tool seperti RsaCtfTool sebagai subprocess
- Chinese Remainder Theorem reconstruction untuk multi-prime RSA

**6.2 Symmetric Cipher Enhancement**
- AES-ECB penguin attack: deteksi ECB mode dari pola blok yang berulang
- CBC bit flipping attack untuk modify plaintext tanpa key
- Stream cipher keystream recovery dari two-time-pad
- Salsa20/ChaCha20 nonce reuse detection
- RC4 key bias detection dan partial key recovery

**6.3 Hash Analysis**
- Hash length extension attack untuk SHA1/SHA256/MD5 tanpa padding
- MD5 collision detection (jika dua file berbeda tapi hash sama)
- Rainbow table integration untuk common hash
- Hash identifier yang lebih komprehensif (50+ hash type)
- Bcrypt/scrypt/argon2 cost parameter analysis

**6.4 Classical Cipher Expansion**
- Playfair cipher solver
- Hill cipher (2x2 dan 3x3 matrix) dengan known-plaintext
- Beaufort cipher
- Autokey cipher
- Running key cipher
- Rail fence cipher (sudah ada? perlu cek)
- Columnar transposition
- Double transposition
- Polybius square variants (ADFGVX, Nihilist)
- Bacon's cipher (binary encoding via font)

**6.5 Custom Encoding Detector**
- Detect dan decode brainfuck, ook!, cow, piet, whitespace language
- Base conversion non-standard (base85, base91, base58, base36, base32hex)
- Custom alphabet substitution: detect jika ciphertext menggunakan custom charset
- Baudot code, punched tape encoding
- T9 predictive text decoding
- Tap code (numbers only cipher)

### 8. INFRASTRUCTURE & UX — Enhancement

**8.1 Plugin Architecture**
Refactor dari satu file monolitik menjadi plugin system. Setiap kategori jadi modul terpisah di folder engine/ yang sudah mulai dibentuk. Benefit: mudah update satu modul tanpa break yang lain, komunitas bisa contribute plugin baru.

**8.2 Result Database**
Simpan hasil analisis ke SQLite database di ~/.raven/results.db. Ini memungkinkan: query hasil analisis lama, track progress per challenge, export report ke JSON/HTML/Markdown.

**8.3 Challenge Context System**
File konfigurasi per-challenge (raven_challenge.yaml) yang bisa berisi: custom flag format, known plaintext hints, challenge category override, wordlist custom. RAVEN baca file ini jika ada di direktori yang sama.

**8.4 Progress Reporting**
Tambahkan progress bar (tqdm-style) untuk operasi yang lama seperti stegseek, hashcat, brute force. Saat ini output langsung tanpa indikasi progress yang jelas.

**8.5 HTML Report Generator**
Setelah analisis selesai, generate laporan HTML yang lengkap dengan: summary eksekutif, semua flag yang ditemukan dengan context, tool yang dijalankan dan hasilnya, file yang diekstrak dengan thumbnail preview untuk gambar, timeline analisis.

**8.6 Docker Image**
Buat Dockerfile yang include semua tools: zsteg, steghide, stegseek, binwalk, volatility3, tshark, john, hashcat, foremost, tesseract, r2, jadx. Eliminasi masalah dependency dan "tool tidak terinstall".

**8.7 CTF Platform Integration**
Deteksi challenge file dari format platform populer (CTFd export, rCTF, PicoCTF), auto-set flag format, track solved status.

## ⚠️ TECHNICAL DEBT YANG PERLU DIADDRESS

Beberapa hal di kode existing yang sebaiknya diperbaiki sebelum menambah fitur baru:

**Memory management** — Beberapa fungsi membaca seluruh file ke memory tanpa streaming, akan crash untuk file >1GB (disk image besar)

**Race condition** — `found_flags_set` dan `FLAG_FOUND` diakses dari multiple thread tanpa locking yang konsisten di semua tempat

**Error recovery** — Banyak except yang hanya `pass` tanpa log, membuat debugging sulit

**Hardcoded paths** — Beberapa path seperti `/usr/share/wordlists/rockyou.txt` hardcoded, sebaiknya configurable

**Engine regeneration** — Engine Python di-overwrite setiap run, tidak perlu jika tidak ada update
